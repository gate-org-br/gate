<g-tab-control>

Within the <g-tab-control> custom web component, anchors (<a>) and buttons (<button>) exhibit distinct behaviors based on the presence and content of subsequent <div> elements. When an anchor or button is accompanied by a non-empty <div>, it functions as an inline tab. Activation of these tabs reveals the associated <div> content, providing an integrated presentation of inline content within the tab structure.

On the other hand, if an anchor or button is followed by an empty <div> or lacks a subsequent <div>, the <g-tab-control> dynamically fetches data based on the attributes specified. For both anchors and buttons, the data-type attribute plays a crucial role. When set to frame, the content is loaded into an <iframe>, while fetch triggers a direct insertion into the <div>. If the data-type attribute is unspecified, it defaults to the type attribute of the <g-tab-control>, which, in turn, defaults to fetch. This design ensures that anchors and buttons without non-empty <div> elements efficiently fetch and display data, enhancing the component's versatility for organizing and presenting content within a tabbed interface.

The <g-tab-control> component in this example creates a tabbed interface with three tabs. The Inline tab displays an inline message when clicked. The Fetch tab fetches a HTML fragment, and the Frame tab fetches an entire document and displays it in an iframe. Icons within the tabs enhance visual appeal. Overall, this example demonstrates the component's versatility for presenting inline content and dynamically fetching data in a tabbed UI with three tabs.

The <g-tab-control> component includes a layout attribute, which, when set to vertical, transforms the default horizontal tab layout into a vertical menu. This attribute alters the visual presentation of the tabs, allowing them to be displayed in a vertical arrangement. By adding the vertical attribute to the <g-tab-control> element, developers can customize the orientation of the tab menu, providing flexibility in the overall design and layout of the component.
