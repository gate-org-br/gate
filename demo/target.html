@populate

Populate the specified target element with a JSON array of objects returned by a request.

Parameters
selector The selector of the target element where the data should be populated.
value This parameter represents the attribute name whose values will be used for the options. If omitted, it defaults to the "value" attribute.
label This parameter represents the attribute name whose values will be displayed as text on the options. If omitted, it defaults to the "label" attribute.
@select

@select

The @select trigger facilitates the selection of an element from a JSON array returned by a request, presenting the choices in a dialog.

Parameters
value Specifies the selector of the element that will receive the value of the selected element. If omitted, the trigger defaults to the first hidden input found within the parent of the target element.
label Specifies the selector of the element that will receive the label of the selected element. If omitted, the trigger defaults to the first text input found within the parent of the target element.

When activated, the @select trigger opens a dialog displaying the available choices, allowing users to make a selection. The chosen value and label are then appropriately assigned to the specified elements, providing a seamless integration of user selections into the document.
@search

target='@select(value, label)'

  • Opens a dialog for the selection of one of the elements returned by the request as a JSON array.
  • The dialog makes a post request to the server sending the search criteria.
  • The value parameter is the selector of the element that will receive the value of the selected element.
  • The value parameter defaults to the first hidden input found on the elements parent.
  • The label parameter is the selector of the element that will receive the label of the selected element.
  • The label parameter defaults to the first text input found on the elements parent.
@pick and @return

@pick

This trigger initiates a dialog that showcases the outcome of a request in the form of an HTML document. When using the this trigger, you provide one or more selectors as parameters. These selectors serve as targets for the values retrieved from the displayed document.

Parameters
selectors... The selectors of the elements that will receive the values retrieved from the displayed document.

@return

For each value in the displayed document, there should be a corresponding @return trigger. These triggers ensure that the values are appropriately placed in their respective @pick trigger elements.

Parameters
values... The values to be returned by the displayed document.

The dialog is closed once a single @return trigger is activated

@dialog
Click to open a fetch dialog or a frame dialog

target='@dialog(type, style)

  • Display the result of of the request on a dialog.
  • If the type is set to fetch, the result is fetched and displayed on a div.
  • If the type is set to frame or ommited, the result is displayed on a iframe.
  • If the style is ommited, the dialog is displayed using 80% of the viewport on large screens and 100% on small ones.
@stack
Click to open a fetch stack or a frame stack

@stack(type)

Display the result of of the request on a modal layer stacked above the current window hiding it completely.

If the type is set to fetch, the result is fetched and displayed on a div.

If the type is set to frame or ommited, the result is displayed on a iframe.

@message

Present the outcome of the request in a message window. The message window automatically closes after the specified duration. If the duration is omitted, the message window will persist until manually closed by the user.

Parameters
duration The duration in milliseconds for which the message will be displayed before automatically closing. If omitted, the message will remain visible until manually closed by the user.
