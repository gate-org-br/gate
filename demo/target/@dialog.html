@dialog

Display the result of the HTTP request in a dialog. The content will be loaded into a <div>; if the type parameter is set to fetch or into an <iframe> if it is omitted or set to frame.

The type and size parameters can be provided in any order for enhanced flexibility. If the type parameter is omitted, it defaults to fetch. When a single size parameter is passed, the dialog adopts equal width and height dimensions. If two size parameters are provided, the first parameter determines the width, and the second parameter determines the height of the dialog. If both the size and height are omitted, the dialog will have 100% width and height with an 80-pixel margin when the screen width is greater than 640 pixels. This versatile approach accommodates various preferences in specifying the appearance of the dialog.

If the trigger element or its parent contains a data-navigator attribute, the dialog will feature a navigator, enabling the user to navigate among all triggers located within it's parent element.
