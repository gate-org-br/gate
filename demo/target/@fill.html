@fill

Simplifies the process of populating input fields with JSON data. Input field selectors are passed as parameters, defaulting to this.parent().search("input[type='hidden']") and this.parent().search("input[type='text']") when omitted.

When handling JSON data, if it's in array format, the values of the target elements are populated with the corresponding array elements. If the JSON data is a generic object, the target elements' values are populated with the object's properties, respecting their order.

Example 1 –

When the action returns a JSON array of objects, the @select target showcases them as cards within a list for user selection. Any attributes beginning with the _ character are omitted from display. Upon user selection, the chosen object is directed to the @fill target. When invoked without parameters, the @fill target populates the value of the first hidden input and the first text input, which are siblings of the trigger element, with the first and second properties of the selected object, respectively.

Example 2 –

When the action returns a JSON array of arrays, the @select target showcases them on a grid for user selection. Any columns with headers beginning with the _ character are omitted from display. Upon user selection, the chosen array is directed to the @fill target. When invoked without parameters, the @fill target populates the value of the first hidden input and the first text input, which are siblings of the trigger element, with the first and second columns of the selected array, respectively.

Example 3 –

On this example, the @fill target will populate the <input id='_ID'>, <input id='Name'>, <input id='Occupation'> and <input id='Birthdate:'> elements respectively with the first, second, third and fourth columns of the selected array from the table.json file.
