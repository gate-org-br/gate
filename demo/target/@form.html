@form

Retrieves a form description from the server and present it to the user for editing. The edited form is then dispatched to the next trigger on the pipeline, if any.

Example 1 –

On this example, the @form trigger fetches the form.json file from the server using the HTTP get method and presents it's as a form to be edited by the user. Upon edition, the edited form is dispatched to the @post trigger. The @post trigger makes a HTTP post with the edited form to form.json and dispatches the result to the @value trigger. The @value trigger updates it's target value with the edited form.
