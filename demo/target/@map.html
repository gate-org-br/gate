@map

The @map target facilitates the transformation of data through the execution of custom JavaScript code before passing it to the next target in the pipeline. This target takes a single parameter: an arrow function. The arrow function is designed to receive the result of the request, process it, and then return the modified data. This allows for flexible and dynamic data processing within the pipeline.

On this example, the @map target fetches objects.json from the server and dispatches only the first 10 elements of the result to the @json-to-html target. The @json-to-html target convert it to HTML and pass the converted information to the @outer-html target. The @outer-html target replaces the <g-callout> with HTML data.
