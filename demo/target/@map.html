@map

The @map trigger facilitates the transformation of data through the execution of custom JavaScript code before passing it to the next trigger in the pipeline. This trigger takes a single parameter: an arrow function. The arrow function is designed to receive the result of the request, process it, and then return the modified data. This allows for flexible and dynamic data processing within the pipeline.

Example 1 –

On this example, the @map trigger fetches objects.json from the server and dispatches only the first 10 elements of the result to the @json-to-html trigger. The @json-to-html trigger convert it to HTML and pass the converted information to the @outer-html trigger. The @outer-html trigger replaces the <g-callout> with HTML data.
