The @select trigger facilitates a dialog for users to choose an element from a JSON array returned by a request. Input elements that will receive the label and value from the trigger can be specified as parameters; otherwise, they default to the first text input and the first hidden input sibling of the trigger element.

Example 1 –

When the action returns an JSON array of objects they are presented as cards on a list for user selection. The first attribute of each object is not displayed as it contains the value to be returned. Upon user selection, the label input receives the first attribute and the value input receives the second one.

Example 2 –

When the result is a JSON array of arrays, they are presented a on a grid for user selection. The first array within the JSON structure serves as the headers for each column in the grid. The first element of each array is not displayed as it contains the value to be returned. Upon user selection, the label input receives the first element of the selected array and the value input receives the second one.

Example 3 –

If the result consists of a JSON array of objects with only label and value attributes, the labels will be displayed on a list for user selection. Upon user selection, the label input and the value input receives the label attribute and the value attribute respectively.

Example 4 –

If the result consists of a JSON array of objects with only label, value and properties attributes, the properties will be displayed as cards on a list for user selection. Upon user selection, the label input and the value input receives the label attribute and the value attribute respectively.
