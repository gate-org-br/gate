@throttle

The @throttle trigger is designed to manage the execution frequency of a specified action. It that another trigger is is executed at most once within the specified time interval in milliseconds. This throttling mechanism prevents the action from being triggered too frequently, promoting efficient and controlled event handling. The target trigger and time interval can be passed as parameters in any order. If the trigger parameter is omitted, it defaults to @none. If the time interval parameter is omitted, it defaults to 1000.

In this example, the "@afterend" trigger is configured to execute at most once every 5 seconds, regardless of how many times the Fetch button is clicked. This setup ensures that the associated action is throttled, preventing rapid and repeated executions within the specified 5-second time interval.
